# CLAUDE.md

## 核心工作规则

### 编写代码前必须向用户确认实现细节

- **禁止假设**：不能假设用户的意图、技术选型、参数设计、命名规范等任何实现细节
- **禁止代替用户做决定**：遇到有多种可行方案的场景时，必须列出选项让用户选择，不能自行决定
- **先澄清需求再编写**：用户描述需求后，如果存在任何不明确的细节，必须先向用户提问确认，待需求明确后方可编写代码
- **允许的操作**：探索代码库、搜索文件、阅读文档等纯研究性任务无需确认

### 业务能力网关 (BCG) 操作规则

在与 BCG 网关交互时（包括但不限于：调用 `/bcg`、查询插件、构建 Graph、执行分析任务），**必须**遵守以下规则：

1. **必须先读 Agent 手册**：每次新会话中，在进行任何 BCG 相关操作之前，必须先阅读 `docs/Agent使用手册.md`，了解正确的 API 调用方式、脚本编写规范、错误码含义。不要凭记忆或猜测行事。

2. **绝对禁止阅读源代码**：遇到错误或不确定用法时，**禁止**阅读 `plugins/`、`core/`、`gateway.py`、`gateway_sdk/` 等源码目录来解决问题。应通过以下方式解决：
   - 查阅 `docs/` 目录下的文档
   - 通过 API 端点获取信息（`GET /plugins`、`GET /plugins/{plugin}/nodes`、`GET /plugins/{plugin}/nodes/{node}`）
   - 根据 Agent 手册的错误码参考判断问题原因

3. **错误排查流程**：当 `/execute` 返回错误时：
   - 先对照手册中的错误码表理解错误含义
   - 用 API 端点确认节点的 input_specs / parameter_specs / output_spec
   - 检查脚本是否符合手册中的三步构建模式（Graph → 节点方法 → output + execute）
   - **禁止**通过阅读源码来排查
