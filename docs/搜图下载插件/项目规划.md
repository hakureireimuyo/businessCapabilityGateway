# Business Capability Gateway + Tag Alignment + Gallery Pipeline

## 项目规划书（Agent可执行版本）

---

# 一、项目目标

构建一个面向 AI Agent 的无状态能力执行系统，实现以下闭环：

> 用户自然语言意图 → Agent语义理解 → tag候选映射 → Agent决策 → gallery-dl执行 → Hydrus入库

系统目标：

* 将自然语言意图转换为**站点可执行 tag 组合**
* 保证所有 tag 必须来自真实站点词表
* 不引入外部状态、记忆或语义数据库
* 所有复杂性集中在 Agent 侧
* Gateway 只负责执行与校验

---

# 二、系统总架构

```
User Input
  ↓
Agent（唯一语义层）
  ↓
Tag Alignment Plugin（逐token候选映射）
  ↓
Agent（逐token选择 + 组合）
  ↓
Gateway（Python sandbox执行）
  ↓
gallery-dl（下载）
  ↓
Hydrus（本地索引）
```

---

# 三、核心设计原则

## 1. 无状态原则（Gateway）

* 不保存任何用户历史
* 不维护任何 session memory
* 每次请求完全独立

---

## 2. 单一语义层（Agent）

Agent负责：

* 解析用户意图
* 拆分语义概念
* 生成 alias set
* 选择最终 tag
* 组合执行计划

系统其他部分不参与语义理解

---

## 3. 逐 token 映射原则（Tag Plugin）

* 每个语义 token 独立处理
* 不允许跨 token 推理
* 不允许组合生成
* 不允许语义扩展

---

## 4. 真实 tag 约束原则

所有输出 tag：

* 必须存在于目标站点
* 必须来源于统计索引
* 必须可直接用于 gallery-dl

---

# 五、Tag Alignment Plugin设计

---

## 5.1 职责

插件仅负责：

* 字符串匹配
* fuzzy matching
* 频次排序
* 返回候选tag集合

---

## 5.2 禁止行为

* 不做语义拆解
* 不做tag组合
* 不做站点策略判断
* 不做跨token分析

---

## 5.3 scoring模型

```
score = similarity_score * α + log(count + 1) * β
```

---

## 5.4 输出结构

```
TagCandidateSet
```

包含：

* tag
* score
* count

---

# 六、Gateway设计（Business Capability Gateway）

---

## 6.1 职责

Gateway仅负责：

* Python脚本执行
* sandbox安全控制
* Node调用校验
* 输出收集

---

## 6.2 禁止行为

* 不理解语义
* 不生成tag
* 不做策略决策
* 不维护状态

---

## 6.3 Sandbox规则

* AST禁止：import / eval / exec / open
* restricted builtins
* timeout execution (10s)

---

## 6.4 Graph执行模型

Agent通过SDK构建：

```python
g = Graph(plugin="tag_alignment")

result = g.tag_query(
    token="双马尾",
    aliases=["twintails", "twin tails"]
)

g.output(result)

final = g.execute()
```

# 九、系统关键约束（必须严格执行）

---

## 1. Tag DB约束

* 只处理单token
* 不做组合
* 不做语义扩展
* 不做站点策略

---

## 2. Agent约束

* 唯一语义解释层
* 唯一决策层
* 可自由生成alias
* 可自由组合tag

---

## 3. Gateway约束

* 只执行Python
* 不参与逻辑
* 不保存状态

---

# 十、完整执行流程

```
1. User Input
2. Agent:
   - semantic understanding
   - token extraction
   - alias generation

3. For each token:
   → Tag Alignment Plugin
   → ranked candidates

4. Agent:
   → select per token tag
   → compose final tag set

5. Gateway:
   → execute graph

6. gallery-dl:
   → download

7. Hydrus:
   → store
```

---

# 十一、系统本质定义

该系统是：

> 一个以 Agent 为唯一语义层的跨站点 tag 映射与执行系统，通过逐 token lexical retrieval 将自然语言意图转换为站点可执行 tag 组合，并由无状态执行内核完成确定性任务执行。

---

# 十二、最关键工程边界

必须严格保持：

* 语义只存在于 Agent
* Tag DB只做排序
* Gateway只做执行
* 所有组合逻辑只在 Agent

---

如果你下一步要做代码实现，优先级最高的是：

1. Tag index schema（倒排 + count）
2. fuzzy matching函数
3. Agent → Graph SDK接口定义

这三项决定系统是否可落地运行。
