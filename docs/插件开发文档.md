## 一、设计哲学

### 1.1 一句话

**Node 不再分类型（Source/Transform/Sink），只声明协议（我吃什么 → 我产出什么 → 我需要什么参数）。**

### 1.2 核心转变

```
旧思维：节点是什么类型 → 节点能放在管道的什么位置 → 怎么按顺序执行
新思维：节点有什么规范 → 节点之间是否兼容         → 怎么按依赖执行
```

一个节点在图中是"入口"还是"转换"还是"终点"，不由它的类别标签决定，而由它在这个特定图中的位置决定。一个有 `input_specs` 的节点，如果这次不连任何上游，它就是事实上的入口。

### 1.3 四大协议层

```
Layer 4: 组合协议 — Graph 构建、边连接、输出声明    ← Agent 视角
Layer 3: 类型协议 — ArtifactType 层次、类型兼容      ← 你需要定义
Layer 2: 能力协议 — Node Spec 的定义与暴露           ← 你需要声明
Layer 1: 执行协议 — 依赖解析、并行调度                ← 网关负责
```

作为插件开发者，你主要工作在 **Layer 2** 和 **Layer 3**：定义 Artifact 类型层次，声明 Node 的协议规范，实现 Node 的业务逻辑。

## 二、架构分层

### 2.1 插件内部三层

```
插件内部架构:

  nodes/          ← Node 层（协议实现 + 对外暴露）
    ├── 声明 input_specs / output_spec / parameter_specs
    ├── 实现 execute(inputs, params, context) → Artifact
    └── 调用 Service 层执行业务逻辑
  services/       ← Service 层（业务逻辑，纯计算）
    ├── 接收原始数据结构（不是 Artifact）
    ├── 执行统计、评分、分析等算法
    └── 返回普通 dict/list 等 Python 数据结构
  repository/     ← Repository 层（数据访问）
    ├── 定义数据模型（dataclass）
    ├── 实现数据查询、过滤、聚合
    └── 管理模拟数据或真实数据源
```

**严格分离**：
- Node 只关心协议（解包 Artifact → 调用 Service → 包装为 Artifact）
- Service 只关心业务（接收原始数据 → 计算 → 返回原始结果）
- Repository 只关心数据（存储 → 查询 → 返回数据模型）
- Node 不直接接触 Repository（除非是入口节点需要获取初始数据）
- Service 不接触 Artifact（它不知道自己在被谁调用）

### 2.2 目录结构

```
plugins/<plugin_name>/
├── plugin.py              # 入口：register() 函数
├── artifact_types.py      # Artifact 类型层次定义
├── nodes/                 # Node 实现（不分 source/transform/sink）
│   ├── __init__.py
│   ├── search_nodes.py    # 数据获取节点
│   ├── filter_nodes.py    # 数据转换节点
│   ├── analysis_nodes.py  # 分析节点
│   └── output_nodes.py    # 格式化输出节点
├── services/              # 业务逻辑
│   ├── __init__.py
│   └── market_service.py
└── repository/            # 数据访问
    ├── __init__.py
    └── product_repository.py
```

## 三、开发步骤

### 步骤一：定义 Artifact 类型层次

在 `plugins/<plugin_name>/artifact_types.py` 中，按业务语义定义类型树。

```python
from core.protocol.artifact import ArtifactType


# ---- 原始数据层 ----
class ProductCollection(ArtifactType):
    """商品数据集合"""
    pass

class FilteredProductCollection(ProductCollection):
    """过滤后的商品集合（ProductCollection 的子类型）"""
    pass


# ---- 指标层 ----
class SalesMetrics(ArtifactType):
    """销售统计指标"""
    pass

class ReviewMetrics(ArtifactType):
    """评论统计指标"""
    pass

class MarketAnalysis(ArtifactType):
    """市场分析结果"""
    pass


# ---- 聚合/输出层 ----
class MarketSignal(ArtifactType):
    """聚合市场信号"""
    pass

class ChartData(ArtifactType):
    """图表可视化数据"""
    pass

class ReportData(ArtifactType):
    """Markdown 报告数据"""
    pass
```

**类型层次设计原则：**

| 原则 | 说明 |
|------|------|
| 子类型 = "是一种更具体的父类型" | `FilteredProductCollection` 是更具体的 `ProductCollection` |
| 名称反映业务语义 | `MarketSignal`，不是 `DictData` |
| 独立业务维度用平级类型 | `SalesMetrics` 和 `ReviewMetrics` 是不同维度 |
| 类型层次要浅 | 通常不超过 2 层深度 |

### 步骤二：实现每个 Node

每个 Node 继承 `core.protocol.node.Node`，声明以下协议属性，并实现 `execute` 方法：

```python
from core.protocol.node import Node
from core.protocol.artifact import Artifact, InputSpec, OutputSpec, ParameterSpec
from core.runtime.context import ExecutionContext
from typing import Any


class MyNode(Node):
    # === 协议声明（类属性） ===

    name = "my_node"              # 插件内唯一标识（英文小写+下划线）
    plugin = "my_plugin"          # 所属插件名
    description = "One-line English description of what this node does"

    input_specs = {
        "input_name": InputSpec(
            name="input_name",
            artifact_type=SomeArtifactType,   # 期望的 ArtifactType 子类
            required=True,                     # 是否必须有上游边
            description="What this input represents",
        ),
    }

    output_spec = OutputSpec(
        key="output_key",                     # 产出在 artifacts 中的存储键
        artifact_type=SomeArtifactType,       # 产出的 Artifact 类型
        description="What this output represents",
    )

    parameter_specs = {
        "param_name": ParameterSpec(
            name="param_name",
            param_type=str,                    # Python 类型: str | int | float | bool
            required=False,                    # 是否必填
            default="default_value",           # 默认值（非必填时）
            description="What this parameter controls",
        ),
    }

    # === 业务实现 ===

    def execute(
        self,
        inputs: dict[str, Artifact],
        params: dict[str, Any],
        context: ExecutionContext,
    ) -> Artifact:
        # 1. 从 inputs 中提取上游数据
        data = inputs["input_name"].data

        # 2. 从 params 中读取配置（默认值已由网关填充）
        option = params.get("param_name", "default")

        # 3. 执行你的业务逻辑
        result = do_something(data, option)

        # 4. 包装为 Artifact 返回
        return Artifact(
            key=self.output_spec.key,
            type=self.output_spec.artifact_type,
            data=result,
            produced_by=self.name,
            metadata={"count": len(result) if hasattr(result, '__len__') else 1},
        )
```

#### 2.1 无输入节点（图入口）

```python
class KeywordSearchNode(Node):
    name = "keyword_search"
    plugin = "amazon"
    description = "Search products by keyword"

    input_specs = {}   # 空字典 — 不需要上游

    output_spec = OutputSpec(
        key="products",
        artifact_type=ProductCollection,
        description="Matching products",
    )

    parameter_specs = {
        "keyword": ParameterSpec("keyword", str, required=True,
                                 description="Search keyword"),
        "limit": ParameterSpec("limit", int, required=False, default=50),
    }

    def __init__(self):
        self._repository = ProductRepository()  # 入口节点可以直接持有 Repository

    def execute(self, inputs, params, context) -> Artifact:
        keyword = params["keyword"]
        limit = params.get("limit", 50)
        products = self._repository.search_by_keyword(keyword)
        if limit:
            products = products.limit(limit)

        return Artifact(
            key="products",
            type=ProductCollection,
            data=products,
            produced_by=self.name,
            metadata={"count": len(products), "keyword": keyword},
        )
```

#### 2.2 转换节点（同类型输入→同类型输出）

```python
class FilterNode(Node):
    name = "filter"
    plugin = "amazon"
    description = "Filter products by multiple conditions (AND semantics)"

    input_specs = {
        "products": InputSpec(
            name="products",
            artifact_type=ProductCollection,    # 接受 ProductCollection
            required=True,
        ),
    }

    output_spec = OutputSpec(
        key="filtered_products",
        artifact_type=FilteredProductCollection,  # 产出子类型
    )

    parameter_specs = {
        "price_gte": ParameterSpec("price_gte", float, required=False),
        "rating_gte": ParameterSpec("rating_gte", float, required=False),
    }

    def execute(self, inputs, params, context) -> Artifact:
        products = inputs["products"].data     # 原始数据对象（不是 Artifact）

        if "price_gte" in params:
            products = products.filter_price_gte(params["price_gte"])
        if "rating_gte" in params:
            products = products.filter_review_rating_gte(params["rating_gte"])

        return Artifact(
            key="filtered_products",
            type=FilteredProductCollection,
            data=products,
            produced_by=self.name,
            metadata={"count": len(products)},
        )
```

**关键**：`FilteredProductCollection` 是 `ProductCollection` 的子类型。这意味着 `filter` 的产出可以流入任何期望 `ProductCollection` 的下游节点（类型兼容规则：`issubclass(FilteredProductCollection, ProductCollection) == True`）。

#### 2.3 分析节点（数据→指标）

```python
class SalesAnalysisNode(Node):
    name = "sales_analysis"
    plugin = "amazon"
    description = "Statistical analysis of sales data"

    input_specs = {
        "products": InputSpec(
            name="products",
            artifact_type=ProductCollection,
            required=True,
        ),
    }

    output_spec = OutputSpec(
        key="sales_metrics",
        artifact_type=SalesMetrics,
    )

    parameter_specs = {}   # 无参数

    def execute(self, inputs, params, context) -> Artifact:
        products = inputs["products"].data

        if len(products) == 0:
            result = {"total_sales": 0, "avg_sales": 0.0, "product_count": 0}
        else:
            sales = [p.monthly_sales for p in products]
            result = {
                "total_sales": sum(sales),
                "avg_sales": round(sum(sales) / len(sales), 1),
                "max_sales": max(sales),
                "product_count": len(products),
            }

        return Artifact(
            key="sales_metrics",
            type=SalesMetrics,
            data=result,
            produced_by=self.name,
        )
```

#### 2.4 多输入汇合节点

```python
class MarketScoreNode(Node):
    name = "market_score"
    plugin = "amazon"
    description = "Compute aggregate score from sales and review metrics"

    input_specs = {
        "sales": InputSpec(
            name="sales",
            artifact_type=SalesMetrics,         # 第一个输入：销售指标
            required=True,
        ),
        "reviews": InputSpec(
            name="reviews",
            artifact_type=ReviewMetrics,        # 第二个输入：评论指标
            required=True,
        ),
    }

    output_spec = OutputSpec(
        key="market_signal",
        artifact_type=MarketSignal,
    )

    parameter_specs = {
        "method": ParameterSpec("method", str, required=False, default="weighted"),
    }

    def execute(self, inputs, params, context) -> Artifact:
        # 从两个不同类型的输入中提取数据
        sales_data = inputs["sales"].data
        review_data = inputs["reviews"].data
        method = params.get("method", "weighted")

        total_sales = sales_data.get("total_sales", 0)
        avg_rating = review_data.get("avg_rating", 0)

        # 计算综合评分
        sales_norm = min(100, int(total_sales / 100)) if total_sales > 0 else 0
        rating_norm = int(avg_rating / 5.0 * 100) if avg_rating > 0 else 0

        if method == "simple":
            signal = int((sales_norm + rating_norm) / 2)
        else:
            signal = int(sales_norm * 0.4 + rating_norm * 0.6)

        return Artifact(
            key="market_signal",
            type=MarketSignal,
            data={
                "market_signal_score": signal,
                "sales_contribution": sales_norm,
                "rating_contribution": rating_norm,
            },
            produced_by=self.name,
        )
```

#### 2.5 通用消费节点（接受任意 Artifact）

当你想做一个通用的格式化/输出节点时，可以将 `input_specs` 的 `artifact_type` 设为 `ArtifactType`（基类）。由于所有 ArtifactType 都是 `ArtifactType` 的子类，任何 Artifact 都可以流入：

```python
from core.protocol.artifact import ArtifactType


class JSONOutputNode(Node):
    name = "json_output"
    plugin = "amazon"
    description = "Output any artifact as raw JSON"

    input_specs = {
        "data": InputSpec(
            name="data",
            artifact_type=ArtifactType,    # 接受任意类型
            required=True,
        ),
    }

    output_spec = OutputSpec(
        key="json",
        artifact_type=JSONData,
    )

    parameter_specs = {}

    def execute(self, inputs, params, context) -> Artifact:
        source = inputs["data"]
        return Artifact(
            key="json",
            type=JSONData,
            data=source.data,               # 透传原始数据
            produced_by=self.name,
        )
```

## 四、实现 Service 层

Service 层负责**纯业务逻辑**。它不接触 Artifact、不接触 ExecutionContext，只接收和返回普通 Python 数据结构。

```python
# plugins/amazon/services/market_service.py

from plugins.amazon.repository.product_repository import ProductCollection


class MarketService:
    """市场分析服务 — 统计、评分、算法"""

    @staticmethod
    def analyze_market(products: ProductCollection) -> dict:
        """计算市场规模、均价、竞争评分"""
        if len(products) == 0:
            return {"market_size": 0, "avg_price": 0.0, "competition_score": 0}

        seller_count = len(products)
        total_reviews = sum(p.review_count for p in products)
        avg_reviews = total_reviews / seller_count if seller_count > 0 else 0

        # 竞争评分逻辑
        if seller_count <= 3:
            competition_score = 15
        elif avg_reviews < 100:
            competition_score = 25
        elif avg_reviews < 500:
            competition_score = 50
        elif avg_reviews < 1000:
            competition_score = 70
        else:
            competition_score = 90

        # 低评论卖家过半 → 蓝海市场
        low_review_count = sum(1 for p in products if p.review_count < 100)
        if low_review_count / seller_count > 0.5:
            competition_score = max(10, competition_score - 20)

        return {
            "market_size": seller_count,
            "avg_price": products.avg_price(),
            "competition_score": min(100, competition_score),
            "total_monthly_sales": products.total_sales(),
        }

    @staticmethod
    def find_opportunities(products: ProductCollection, max_review: int) -> dict:
        """寻找低竞争高潜力的产品机会"""
        opportunities = []
        for p in products:
            if p.review_count < max_review:
                score = min(100, int(
                    (p.review_rating / 5.0) * 40 +
                    (min(p.monthly_sales, 2000) / 2000) * 40 +
                    (1 - p.review_count / max_review) * 20
                ))
                if score > 30:
                    opportunities.append({
                        "asin": p.asin,
                        "title": p.title,
                        "price": p.price,
                        "review_count": p.review_count,
                        "review_rating": p.review_rating,
                        "monthly_sales": p.monthly_sales,
                        "opportunity_score": score,
                    })

        opportunities.sort(key=lambda x: x["opportunity_score"], reverse=True)
        return {"opportunity_count": len(opportunities), "opportunities": opportunities[:10]}
```

## 五、实现 Repository 层

Repository 层负责**数据访问**。定义数据模型（dataclass）和数据查询方法。

```python
# plugins/amazon/repository/product_repository.py

from dataclasses import dataclass


@dataclass
class Product:
    """产品数据模型"""
    asin: str
    title: str
    keyword: str
    price: float
    review_count: int
    review_rating: float
    category: str
    monthly_sales: int


class ProductCollection:
    """产品集合 — 内存数据容器，支持链式过滤"""

    def __init__(self, products: list[Product] | None = None):
        self._products: list[Product] = products or []

    def __len__(self) -> int:
        return len(self._products)

    def __iter__(self):
        return iter(self._products)

    # 过滤方法（每个返回新的 ProductCollection，支持链式调用）
    def filter_price_gte(self, min_price: float) -> "ProductCollection":
        return ProductCollection([p for p in self._products if p.price >= min_price])

    def filter_review_rating_gte(self, min_rating: float) -> "ProductCollection":
        return ProductCollection([p for p in self._products if p.review_rating >= min_rating])

    # 聚合方法
    def avg_price(self) -> float:
        if not self._products:
            return 0.0
        return round(sum(p.price for p in self._products) / len(self._products), 2)

    def total_sales(self) -> int:
        return sum(p.monthly_sales for p in self._products)

    def limit(self, n: int) -> "ProductCollection":
        return ProductCollection(self._products[:n])


class ProductRepository:
    """产品数据仓库 — 负责数据获取"""

    def __init__(self):
        self._products: list[Product] = []  # 初始化时加载数据

    def get_all(self) -> ProductCollection:
        return ProductCollection(list(self._products))

    def search_by_keyword(self, keyword: str) -> ProductCollection:
        keyword_lower = keyword.lower()
        matched = [
            p for p in self._products
            if keyword_lower in p.keyword.lower() or keyword_lower in p.title.lower()
        ]
        return ProductCollection(matched)
```

## 六、创建插件入口

在 `plugins/<plugin_name>/plugin.py` 中写一个 `register()` 函数：

```python
"""MyPlugin — entry point"""

from core.registry.node_registry import register_nodes
from plugins.my_plugin.nodes.search_nodes import KeywordSearchNode
from plugins.my_plugin.nodes.filter_nodes import FilterNode
from plugins.my_plugin.nodes.analysis_nodes import MarketAnalysisNode, SalesAnalysisNode
from plugins.my_plugin.nodes.output_nodes import ChartOutputNode, JSONOutputNode


def register():
    """Register all plugin nodes with the gateway."""
    register_nodes("my_plugin", [
        # 图入口
        KeywordSearchNode(),
        # 数据转换
        FilterNode(),
        # 分析
        MarketAnalysisNode(),
        SalesAnalysisNode(),
        # 输出
        ChartOutputNode(),
        JSONOutputNode(),
    ])
```

**约束**：
- `register()` 函数名固定，不接受参数
- 必须在模块顶层（不是类方法或嵌套函数）
- 每个 Node 实例化一次即可（Node 必须是无状态的，共享实例安全）
- `plugin` 属性必须与 `register_nodes` 的第一个参数一致

## 七、Node 开发规范

### 7.1 Node 约束

| 约束 | 说明 |
|------|------|
| 无状态 | 每个 `execute()` 调用必须独立，不依赖前一次调用的状态 |
| 不调用其他 Node | Node 之间不能互相调用，只能通过 Graph 边连接 |
| 不访问外部状态 | 除了 Repository（数据访问），不读文件、不发网络请求 |
| 不修改 context | 不在 `execute()` 中写 `context.artifacts` 或 `context.outputs`（由网关负责） |
| 单一输出 | 每个 Node 只有一个 `output_spec`，多输出需求通过拆分 Node 实现 |

### 7.2 execute() 签名

```python
def execute(
    self,
    inputs: dict[str, Artifact],   # 已解析的输入 Artifact，key 与 input_specs 的 key 一致
    params: dict[str, Any],        # 已解析的参数（必填已校验，默认值已填充）
    context: ExecutionContext,     # 全局上下文（只读 metadata，不写入）
) -> Artifact:                     # 返回一个 Artifact
```

### 7.3 命名规范

| 对象 | 规范 | 示例 |
|------|------|------|
| `Node.name` | 英文小写+下划线 | `keyword_search`, `market_analysis` |
| `Node.plugin` | 英文小写，无空格 | `amazon`, `tiktok_shop` |
| `InputSpec.name` | 英文小写 | `products`, `sales`, `reviews` |
| `OutputSpec.key` | 英文小写 | `products`, `sales_metrics`, `market_signal` |
| `ParameterSpec.name` | 英文小写+下划线 | `price_gte`, `max_review` |
| `ArtifactType` 类名 | PascalCase | `ProductCollection`, `SalesMetrics` |

### 7.4 Artifact.metadata

每次返回 Artifact 时，建议填写 `metadata` 字典：

```python
return Artifact(
    key=self.output_spec.key,
    type=ProductCollection,
    data=products,
    produced_by=self.name,
    metadata={
        "count": len(products),          # 数据量
        "keyword": keyword,              # 关键参数（便于追踪）
    },
)
```

`metadata` 不参与业务逻辑，但有助于调试和 Agent 理解数据来源。

## 八、Graph 校验规则

你的 Node Spec 声明会被 GraphValidator 在七层校验中使用。以下是各层与你声明的对应关系：

| 层 | 检查什么 | 你的声明对应 | 常见错误原因 |
|----|---------|-------------|-------------|
| 1 | 节点是否存在 | `plugin` + `name` | `name` 拼写错误或未注册 |
| 2 | 参数是否合法 | `parameter_specs` | 参数类型不匹配、必填参数缺失、传了未声明参数 |
| 3 | 输入是否完备 | `input_specs` 中 `required=True` | 必填输入没有上游边 |
| 4 | 类型是否兼容 | `input_specs[].artifact_type` + `output_spec.artifact_type` | 上游产出类型不满足下游期望 |
| 5 | 是否无环 | （图结构） | 边形成了环 |
| 6 | 输出是否合法 | （图结构） | outputs 引用不存在的 node_id |
| 7 | 是否单插件 | `plugin` 属性 | 图中混用了不同插件的节点 |

### 9.1 目录结构

```
plugins/tiktok/
├── plugin.py
├── artifact_types.py
├── nodes/
```

### 9.3 repository/video_repository.py

```python
from dataclasses import dataclass


@dataclass
class Video:
    video_id: str
    author: str
    description: str
    hashtags: list[str]
    views: int
    likes: int
    comments: int
    shares: int
    duration_sec: int


class VideoCollection:
    def __init__(self, videos: list[Video] | None = None):
        self._videos: list[Video] = videos or []

    def __len__(self): return len(self._videos)
    def __iter__(self): return iter(self._videos)

    def filter_min_likes(self, min_likes: int) -> "VideoCollection":
        return VideoCollection([v for v in self._videos if v.likes >= min_likes])

    def filter_hashtag(self, tag: str) -> "VideoCollection":
        return VideoCollection([v for v in self._videos if tag in v.hashtags])

    def avg_engagement_rate(self) -> float:
        if not self._videos:
            return 0.0
        rates = [(v.likes + v.comments + v.shares) / max(v.views, 1) for v in self._videos]
        return round(sum(rates) / len(rates), 4)
```

### 9.4 services/tiktok_service.py

```python
from plugins.tiktok.repository.video_repository import VideoCollection


class TikTokService:

    @staticmethod
    def analyze_engagement(videos: VideoCollection) -> dict:
        if len(videos) == 0:
            return {"total_videos": 0, "avg_likes": 0, "avg_comments": 0,
                    "avg_shares": 0, "avg_engagement_rate": 0.0}

        n = len(videos)
        return {
            "total_videos": n,
            "total_views": sum(v.views for v in videos),
            "avg_likes": round(sum(v.likes for v in videos) / n, 1),
            "avg_comments": round(sum(v.comments for v in videos) / n, 1),
            "avg_shares": round(sum(v.shares for v in videos) / n, 1),
            "avg_engagement_rate": videos.avg_engagement_rate(),
        }

    @staticmethod
    def analyze_trends(videos: VideoCollection) -> dict:
        if len(videos) == 0:
            return {"trending_hashtags": [], "top_authors": []}

        # 标签频次统计
        tag_counts: dict[str, int] = {}
        for v in videos:
            for tag in v.hashtags:
                tag_counts[tag] = tag_counts.get(tag, 0) + 1
        trending = sorted(tag_counts.items(), key=lambda x: x[1], reverse=True)[:5]

        # 作者影响力统计
        author_stats: dict[str, dict] = {}
        for v in videos:
            if v.author not in author_stats:
                author_stats[v.author] = {"videos": 0, "total_views": 0, "total_likes": 0}
            author_stats[v.author]["videos"] += 1
            author_stats[v.author]["total_views"] += v.views
            author_stats[v.author]["total_likes"] += v.likes

        top_authors = sorted(
            author_stats.items(),
            key=lambda x: x[1]["total_likes"],
            reverse=True,
        )[:3]

        return {
            "trending_hashtags": [{"tag": t, "count": c} for t, c in trending],
            "top_authors": [{"author": a, **s} for a, s in top_authors],
        }
```

### 9.5 nodes/search_nodes.py

```python
from core.protocol.node import Node
from core.protocol.artifact import Artifact, InputSpec, OutputSpec, ParameterSpec
from core.runtime.context import ExecutionContext
from plugins.tiktok.artifact_types import VideoCollection
from plugins.tiktok.repository.video_repository import VideoRepository
from typing import Any


class HashtagSearchNode(Node):
    name = "hashtag_search"
    plugin = "tiktok"
    description = "Search TikTok videos by hashtag"

    input_specs = {}

    output_spec = OutputSpec(
        key="videos",
        artifact_type=VideoCollection,
        description="Matching videos",
    )

    parameter_specs = {
        "hashtag": ParameterSpec("hashtag", str, required=True,
                                 description="Hashtag to search (without #)"),
        "limit": ParameterSpec("limit", int, required=False, default=50),
    }

    def __init__(self):
        self._repository = VideoRepository()

    def execute(
        self, inputs: dict[str, Artifact], params: dict[str, Any],
        context: ExecutionContext,
    ) -> Artifact:
        hashtag = params["hashtag"]
        limit = params.get("limit", 50)
        videos = self._repository.search_by_hashtag(hashtag)
        if limit:
            videos = VideoCollection(list(videos)[:limit])

        return Artifact(
            key="videos", type=VideoCollection, data=videos,
            produced_by=self.name,
            metadata={"count": len(videos), "hashtag": hashtag},
        )
```

## 十、设计原则与检查清单

### 开发前自检

- [ ] 所有 Artifact 类型是否都继承自 `ArtifactType`？
- [ ] 每个 Node 的 `name` 是否在插件内唯一？
- [ ] 每个 Node 的 `plugin` 属性是否与 `register_nodes` 的参数一致？
- [ ] `output_spec.key` 是否描述了产出的业务含义？
- [ ] `input_specs.artifact_type` 是否使用了最合适的抽象层次？（太具体会限制可组合性，太宽泛会失去类型安全）
- [ ] `parameter_specs.param_type` 是否与实际 `params.get()` 的用法一致？
- [ ] `execute()` 是否只通过 `inputs` 和 `params` 获取输入，通过返回值产出输出？
- [ ] 是否避免在 `execute()` 中修改 `context`？

### 不要做的事

- **不要给 Node 分类**（没有 `node_type` 字段，不需要分 source/transform/sink 目录）
- **不要在 Node 之间直接调用**（所有连接通过 Graph 边表达）
- **不要在 Service 层接触 Artifact**（Service 接收和返回纯数据）
- **不要修改网关核心代码**（新增插件不需要改 `core/` 下的任何文件）
- **不要跨插件引用**（一个 Node 只能属于一个插件）

### 推荐做的事

- 类型层次保持浅层（1-2 层），用语义清晰的类名
- 每个 Node 的 `description` 写清楚一句话英文说明
- 每个 `InputSpec`、`OutputSpec`、`ParameterSpec` 填写 `description`
- Artifact 的 `metadata` 记录关键信息（数据量、关键参数）
- Service 方法用 `@staticmethod`（无状态）
- Repository 数据模型用 `@dataclass`
