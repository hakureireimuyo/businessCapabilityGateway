# 插件开发协议

本文档定义插件开发者必须遵守的全部规范。阅读本文档后，你可以为任何数据库创建业务能力插件。

---

## 1. 核心原则

```
插件负责业务，网关负责协议
```

- 插件**只管**实现业务能力（查库、统计、分析、评分）
- 插件**不管**协议路由、DSL 解析、请求分发
- 新增插件**不修改网关任何一行代码**

---

## 2. 目录结构

```
plugins/
└── <plugin_name>/              # 插件名 = 命名空间（小写英文，用下划线分隔）
    ├── __init__.py
    ├── plugin.py               # 【必需】插件入口，调用 register()
    ├── nodes/                  # 【必需】Node 定义
    │   ├── __init__.py
    │   ├── source_nodes.py     # Source Node
    │   ├── transform_nodes.py  # Transform Node
    │   └── sink_nodes.py       # Sink Node
    ├── services/               # 【可选】业务逻辑
    │   └── __init__.py
    └── repository/             # 【可选】数据访问
        └── __init__.py
```

---

## 3. plugin.py 入口规范

`plugin.py` 是插件唯一的入口文件，必须包含 `register()` 函数：

```python
"""<插件名> 插件入口"""

from core.node_registry import register_nodes
from plugins.<plugin_name>.nodes.source_nodes import YourSourceNode
from plugins.<plugin_name>.nodes.transform_nodes import YourFilterNode
from plugins.<plugin_name>.nodes.sink_nodes import YourResultNode


def register():
    """向网关注册本插件的所有 Node"""
    register_nodes("<plugin_name>", [
        YourSourceNode(),
        YourFilterNode(),
        YourResultNode(),
    ])
```

**约束**：
- `register()` 函数签名必须无参数
- 必须使用 `register_nodes(plugin_name, node_list)` 注册
- `plugin_name` 必须与目录名一致

## 4. Node 开发规范

### 4.1 Node 基类

```python
from core.node import Node, NodeType
from core.execution_context import ExecutionContext


class YourNode(Node):
    # ---- 必须定义的类属性 ----
    name: str = "node_name"           # 英文小写+下划线，Agent 调用时使用
    description: str = "..."           # 一句英文描述
    node_type: str = NodeType.SOURCE   # SOURCE | TRANSFORM | SINK
    parameters: dict = { ... }         # 参数 schema
    input_schema: dict | None = None   # 输入类型（Source 为 None）
    output_schema: dict | None = None  # 输出 schema

    # ---- 必须实现的方法 ----
    def execute(self, context: ExecutionContext) -> ExecutionContext:
        ...
        return context
```

### 4.2 三种节点类型

| 类型 | 常量 | 输入 | 输出 | 参数语义 | 管道位置 |
|------|------|------|------|----------|----------|
| **Source** | `NodeType.SOURCE` | 无（从数据库读取） | 数据集（存 context.data） | 控制数据获取范围 | 管道第一个 |
| **Transform** | `NodeType.TRANSFORM` | 数据集（从 context.data 读取） | 数据集（写回 context.data） | 控制数据筛选/排序 | 管道中间 |
| **Sink** | `NodeType.SINK` | 数据集（从 context.data 读取） | 最终结果（写入 context.result） | 控制分析方法，不控制数据范围 | 管道最后一个 |

### 4.3 Source Node 示例

```python
class SearchNode(Node):
    name = "search_products"
    description = "Search products by keyword"
    node_type = NodeType.SOURCE
    parameters = {
        "keyword": {"type": "string", "required": True, "description": "Search keyword"},
        "limit": {"type": "integer", "required": False, "default": 50, "description": "Max results"},
    }
    # 注意：Source 无 input_schema
    output_schema = {"type": "ProductCollection"}

    def execute(self, context: ExecutionContext) -> ExecutionContext:
        keyword = context.current_params["keyword"]
        limit = int(context.current_params.get("limit", 50))

        products = self.repository.search(keyword, limit)

        context.data = products          # 将结果集放入 context.data
        context.metadata["count"] = len(products)
        return context
```

**关键约定**：
- 从数据库读取数据，构造内存集合对象
- 将集合对象赋值给 `context.data`
- **绝不直接从数据库返回原始 dict / list / SQL 结果**

### 4.4 Transform Node 示例

```python
class FilterNode(Node):
    name = "filter"
    description = "Filter products by conditions"
    node_type = NodeType.TRANSFORM
    parameters = {
        "price_gte": {"type": "float", "required": False},
        "review_lt": {"type": "integer", "required": False},
    }
    input_schema = {"type": "ProductCollection"}
    output_schema = {"type": "ProductCollection"}

    def execute(self, context: ExecutionContext) -> ExecutionContext:
        products = context.data           # 从前一节点读取数据

        if "price_gte" in context.current_params:
            products = products.filter(lambda p: p.price >= float(context.current_params["price_gte"]))
        if "review_lt" in context.current_params:
            products = products.filter(lambda p: p.reviews < int(context.current_params["review_lt"]))

        context.data = products           # 写回 context.data
        return context
```

**关键约定**：
- 从 `context.data` 读取数据集
- 修改后写回 `context.data`
- 链式过滤优先（`Products.filter_a().filter_b()`）

### 4.5 Sink Node 示例

```python
class AnalysisNode(Node):
    name = "market_analysis"
    description = "Analyze market and return scores"
    node_type = NodeType.SINK
    parameters = {}                      # 无额外参数
    input_schema = {"type": "ProductCollection"}
    output_schema = {                    # 声明输出字段
        "market_size": "integer",
        "avg_price": "float",
        "competition_score": "integer (0-100)",
    }

    def execute(self, context: ExecutionContext) -> ExecutionContext:
        products = context.data

        result = {
            "market_size": len(products),
            "avg_price": products.avg_price(),
            "competition_score": self._calculate_score(products),
        }

        context.result = result           # 写入最终结果
        return context
```

**关键约束**：
- 从 `context.data` 读取数据集
- 将分析结果字典写入 `context.result`
- Sink 的输出必须是**经过计算得出的业务结果**，不能直接把管道中拿到的数据集合原样返回

### 4.6 Sink 参数语义规则

**所有 Sink 都接受管道数据作为输入。Sink 可以声明自己的 parameters，但这些参数只能控制"如何处理输入数据"（分析方法），不能控制"筛选哪些数据"（数据范围）。**

```
┌─────────────────────────────────────────────────────┐
│  过滤 = Transform 的职责                             │
│  分析 = Sink 的职责                                  │
│  二者不可混淆                                       │
└─────────────────────────────────────────────────────┘
```

**✅ 合法的 Sink 参数（分析调优）**：

```python
# 控制分析阈值（不是排除数据，是改变"判断标准"）
"max_review": {"type": "integer", "description": "Max review count threshold for opportunity scoring"}

# 控制输出形式
"top_n": {"type": "integer", "description": "Return only top N results"}

# 控制算法行为
"method": {"type": "string", "description": "Scoring method: weighted | simple"}
```

这些参数的特点是：同样的输入数据，改参数值会产生不同的**分析结论**，不是不同的**数据范围**。

**❌ 非法的 Sink 参数（数据筛选，应放在 Transform）**：

```python
# 以下参数属于数据筛选，不应出现在 Sink 上
"price_gte": ...    # → 交给 filter Transform
"category": ...     # → 交给 filter Transform
"rating_lt": ...    # → 交给 filter Transform
```

**为什么这样设计**：

| 维度 | 说明 |
|------|------|
| 职责清晰 | Transform 管筛选，Sink 管分析，Agent 不会困惑"到底在哪过滤" |
| 无隐喻依赖 | 所有 Sink 都明确依赖 context.data，不存在"暗中自己查数据"的 Sink |
| 可组合性 | Agent 可以在同一个数据集上串联多个不同分析角度 |
| 可测试性 | 分析逻辑的输入完全可控，不依赖隐式数据获取 |

**Agent 视角的决策树**：

```
需要缩小数据范围？ → 加 Transform 节点（filter / sort）
需要改变分析方式？ → 调 Sink 参数
两者都要？        → filter 负责前者 + Sink 参数负责后者
```

## 5. 参数 schema 规范

```python
parameters = {
    "param_name": {
        "type": "string",             # string | integer | float
        "required": True,             # True | False
        "default": default_value,     # 可选，required=False 时生效
        "description": "...",         # 一句英文描述
    }
}
```

网关会自动校验参数类型和必填项。

## 6. 数据载体规范

### 6.1 ExecutionContext

节点间传递的唯一对象：

```python
context.request_id         # str:  请求唯一标识
context.plugin_name        # str:  当前插件名
context.current_params     # dict: 当前节点的参数
context.data               # Any:  中间数据集（Source/Transform 写入，Transform/Sink 读取）
context.result             # dict: 最终结果（仅 SinkNode 写入）
context.metadata           # dict: 附加信息（行数、耗时等）
```

### 6.2 数据集合对象

必须封装为内存对象，提供链式操作方法。**禁止直接暴露 list/dict/cursor**：

```python
class ProductCollection:
    """推荐：支持链式过滤的内存集合"""

    def filter_price_gte(self, min_price: float) -> "ProductCollection": ...
    def filter_review_lt(self, max_review: int) -> "ProductCollection": ...
    def sort_by_sales(self) -> "ProductCollection": ...
    def limit(self, n: int) -> "ProductCollection": ...
    def avg_price(self) -> float: ...
    def __len__(self) -> int: ...
```

## 7. 强制约束清单

| # | 规则 | 违规后果 |
|---|------|---------|
| 1 | Node 是插件暴露给网关的**唯一**单元 | 不得暴露 Repository/Service |
| 2 | Node 必须**无状态**，每次 execute 独立 | 并发 Agent 调用互相污染 |
| 3 | 参数名、节点名使用**英文小写+下划线** | Agent 无法正确生成 BCL |
| 4 | `node_type` 必须与用途匹配 | 管道校验失败 |
| 5 | Sink 输出必须是**经计算/筛选/评分后的业务结果**，不能原样返回 context.data | 违反"Agent 不接触数据"原则 |
| 6 | 数据集合对象**禁止**直接暴露给 Agent，Sink 必须转为 dict | 同上 |
| 7 | Repository **禁止**包含业务逻辑 | 业务规则和数据逻辑混在一起 |
| 8 | 插件**禁止**调用其他插件 | 插件间完全隔离 |
| 9 | 结果对象使用**统一 dict 格式**（由 Sink 定义） | Agent 理解困难 |
| 10 | `register()` 函数必须存在且无参数 | 插件加载失败 |

---

## 8. 新增插件检查清单

- [ ] `plugins/<name>/plugin.py` 存在且包含 `register()` 函数
- [ ] 至少有 1 个 Source + 1 个 Sink
- [ ] 所有 Node 的 `name`/`description`/`parameters` 为英文
- [ ] Source 的 `input_schema` 为 None
- [ ] Transform 的 `input_schema` 和 `output_schema` 类型一致
- [ ] Sink 的 `output_schema` 声明了所有输出字段
- [ ] 数据集合对象封装在 class 中，不暴露原始 list/dict
- [ ] 每个 Node 无实例级可变状态
- [ ] 未在 Repository 中编写业务逻辑
- [ ] 未引用其他插件的模块

## 9. 快速模板

```python
# ========== plugins/my_app/plugin.py ==========
from core.node_registry import register_nodes
from plugins.my_app.nodes.source_nodes import QueryNode
from plugins.my_app.nodes.transform_nodes import FilterNode
from plugins.my_app.nodes.sink_nodes import ReportNode


def register():
    register_nodes("my_app", [
        QueryNode(),
        FilterNode(),
        ReportNode(),
    ])
```

```python
# ========== plugins/my_app/nodes/sink_nodes.py ==========
from core.node import Node, NodeType
from core.execution_context import ExecutionContext


class ReportNode(Node):
    name = "generate_report"
    description = "Generate business report from filtered data"
    node_type = NodeType.SINK
    parameters = {}
    input_schema = {"type": "MyDataCollection"}
    output_schema = {
        "total": "integer",
        "avg": "float",
    }

    def execute(self, context: ExecutionContext) -> ExecutionContext:
        data = context.data
        context.result = {
            "total": len(data),
            "avg": data.average(),
        }
        return context
```
