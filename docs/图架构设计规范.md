\# Business Graph Runtime Architecture



\## 设计目标



本系统不采用传统 Pipeline 架构。



不再使用：



Source -> Transform -> Sink



而采用：



Node + DAG + Artifact



架构。



目标：



\* 支持分支

\* 支持汇聚

\* 支持并行执行

\* 支持多分析器

\* 支持多输出器

\* 支持未来扩展



\---



\# 核心概念



\## Node



Node 是系统中的最小执行单元。



Node 只负责：



\* 读取输入

\* 执行逻辑

\* 生成输出



Node 不关心执行顺序。



Node 不主动调用其他 Node。



Node 只声明：



\* 输入

\* 输出



例如：



KeywordSearchNode



输入：



无



输出：



products



\---



MarketAnalysisNode



输入：



products



输出：



market\_analysis



\---



ChartOutputNode



输入：



market\_analysis



输出：



chart



\---



\# Artifact



Artifact 是节点之间传递的数据。



例如：



products



sales\_analysis



review\_analysis



market\_analysis



chart



report



Artifact 是图中的边。



Node 是图中的点。



\---



\# Graph



Graph 为有向无环图（DAG）。



例如：



products

├── sales\_analysis

├── review\_analysis

└── price\_analysis



sales\_analysis

review\_analysis

price\_analysis

└── market\_score



market\_score

├── chart

└── report



\---



\# Node 定义



Node 必须声明：



id



inputs



outputs



execute()



示例：



Node



id:

market\_analysis



inputs:

\[

"products"

]



outputs:

\[

"market\_analysis"

]



\---



\# ExecutionContext



ExecutionContext 为全局运行上下文。



结构：



datasets



artifacts



outputs



metadata



\---



datasets



原始数据集。



例如：



products



keywords



sales



\---



artifacts



中间分析结果。



例如：



market\_analysis



review\_analysis



competition\_score



\---



outputs



最终输出结果。



例如：



chart



markdown



pdf



json



\---



metadata



运行信息。



例如：



request\_id



plugin



timestamp



execution\_time



\---



\# 执行流程



步骤：



1\. 解析 BCL

2\. 构建 AST

3\. 构建 DAG

4\. 检查循环依赖

5\. 拓扑排序

6\. 执行 Node

7\. 生成 Outputs



\---



\# 执行规则



节点执行条件：



所有输入 Artifact 已存在。



例如：



MarketAnalysis



需要：



products



只有 products 已生成后才能执行。



\---



\# 并行执行



若两个节点无依赖关系：



sales\_analysis(products)



review\_analysis(products)



则允许并行执行。



\---



\# 禁止事项



禁止循环依赖。



例如：



A -> B



B -> C



C -> A



必须在构图阶段报错。



\---

未来支持：



\* 多分析器

\* 多输出器

\* 缓存

\* 增量计算

\* Agent 自动规划

只负责执行图。
